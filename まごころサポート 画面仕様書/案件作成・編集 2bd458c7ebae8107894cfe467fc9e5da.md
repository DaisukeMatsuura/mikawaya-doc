## 🎯 画面の目的

新規のサポート依頼（案件）を作成する。

---

## 📊 表示データ項目

### 基本情報

| 項目名 | データ型 | 必須 | 備考 |
| --- | --- | --- | --- |
| 担当店舗 | Enum | ◯ | セレクト（店舗マスタから選択） |
| お客様名 | Enum | ◯ | セレクト（顧客マスタから選択） |
| 行動種別 | Enum | ◯ | セレクト |
| 取次事項 | String | - | テキストエリア、ツールチップで説明あり |

### 事前情報

| 項目名 | データ型 | 必須 | 備考 |
| --- | --- | --- | --- |
| 訪問予定日 | Date | - | 日付選択 |
| 訪問予定時間 | Time | - | セレクトまたは入力 |
| 受注経路 | Enum | - | セレクト |
| 受注可否（ステータス） | Enum | ◯ | デフォルト：確認中 |
| 担当コンシェルジュ | Enum | - | セレクト（コンシェルジュマスタから選択） |

---

## ⚙️ 機能・操作

| 要素 | 操作 | 動作 |
| --- | --- | --- |
| 担当店舗セレクト | 選択 | 店舗を選択 |
| お客様名セレクト | 選択 | 顧客を選択（検索可能） |
| 行動種別セレクト | 選択 | 行動種別を選択 |
| 取次事項テキストエリア | 入力 | 自由入力 |
| 訪問予定日 | 選択 | カレンダーから日付選択 |
| 訪問予定時間 | 選択/入力 | 時間を選択または入力 |
| 受注経路セレクト | 選択 | 受注経路を選択 |
| 受注可否セレクト | 選択 | ステータスを選択 |
| 担当コンシェルジュセレクト | 選択 | コンシェルジュを選択 |
| キャンセルボタン | クリック | モーダルを閉じる（入力内容は破棄） |
| 下書き保存ボタン | クリック | 下書きとして保存 |
| 保存ボタン | クリック | 案件を作成 |

---

## 🔄 状態パターン

| 状態 | 表示内容 |
| --- | --- |
| 新規作成モード | フォーム空欄 |
| コピー作成モード | コピー元の内容を表示 |
| バリデーションエラー | 該当項目を赤枠 + エラーメッセージ |
| 処理中 | ボタンをローディング表示 |
| 保存完了 | モーダルを閉じる、一覧を更新、トースト通知「案件を作成しました」 |
| 下書き保存完了 | モーダルを閉じる、トースト通知「下書きを保存しました」 |

## 📝 備考

- API: `POST /api/operator/cases`（新規作成）
- API: `POST /api/operator/cases/draft`（下書き保存）
- 案件Noは保存時に自動採番
- 受注可否のデフォルトは「確認中」
- 「案件コピー」から開いた場合、コピー元の内容がプリセットされる
- お客様名セレクトは検索可能（顧客数が多いため）
- 担当店舗を選択すると、その店舗に所属するコンシェルジュのみが担当コンシェルジュに表示される
