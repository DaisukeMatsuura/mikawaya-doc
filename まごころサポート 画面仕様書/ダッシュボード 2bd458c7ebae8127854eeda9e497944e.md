## 🎯 画面の目的

管理者がシステム全体のサマリーデータを一目で把握し、システム運用の健全性を確認できるようにする。

---

## 📊 表示データ項目

### サマリーカード

| 項目名 | データ型 | 備考 |
| --- | --- | --- |
| アクティブユーザー | Number | 全ユーザー種別の合計 |
| 新規案件（本日） | Number | 件数表示 |
| 未対応案件 | Number | 件数表示 |
| 完了報告率（月間） | Percentage | %表示 |

### システム統計

| 項目名 | データ型 | 備考 |
| --- | --- | --- |
| 総案件数（累計） | Number | 全件数 |
| 月間売上 | Number | 円表示 |
| パートナー加盟店数 | Number | アクティブなみ |
| オペレータ数 | Number | アクティブなみ |

### 最近の活動

| 項目名 | データ型 | 備考 |
| --- | --- | --- |
| 操作日時 | DateTime |  |
| 操作者 | String | ユーザー名 |
| 操作内容 | String | 例：案件作成、アカウント編集 |
| 対象 | String | 案件No等 |

---

## ⚙️ 機能・操作

| 要素 | 操作 | 動作 |
| --- | --- | --- |
| ハンバーガーメニュー | クリック | サイドメニューを表示 |
| サマリーカード | クリック | 該当条件で絞り込んだ一覧へ遷移 |
| 統計グラフ | ホバー | 詳細情報をツールチップで表示 |
| 最近の活動行 | クリック | 対象の詳細画面へ遷移 |
| 更新ボタン | クリック | 画面データを再読み込み |

---

## 🔄 状態パターン

| 状態 | 表示内容 |
| --- | --- |
| 初期表示 | ローディングスピナー |
| データあり | サマリー + 統計 + 最近の活動を表示 |
| データなし | 「データがありません。」メッセージ表示 |

| --- | --- |

## 📝 備考

- API: `GET /api/admin/dashboard`
- PC対応（Responsive Web Design）
- 管理者のみアクセス可
- 統計データは日次更新
- 最近の活動は最大10件表示
- 月間売上は計上済みの売上のみをカウント
