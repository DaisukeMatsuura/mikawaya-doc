## 🎯 画面の目的

全案件の一覧を表示し、検索・絞り込みを行い、案件の管理を行う。

---

## 📊 表示データ項目

| 項目名 | データ型 | 必須 | 備考 |
| --- | --- | --- | --- |
| 受付日時 | DateTime | ◯ | ソート可能 |
| 訪問予定日時 | DateTime | - | ソート可能 |
| 案件No | String | ◯ | ソート可能 |
| お客様名 | String | ◯ | ソート可能 |
| サポート種別 | String | ◯ | 掃除/草むしり/片付け/買い物 等 |
| 行動種別 | String | ◯ | 対面 等 |
| ステータス | Enum | ◯ | バッジ表示、ソート可能 |
| 完了日 | DateTime | - | ソート可能 |
| サポート金額 | Number | - | ソート可能 |

---

## ⚙️ 機能・操作

| 要素 | 操作 | 動作 |
| --- | --- | --- |
| 案件検索 | 入力 | 案件No、お客様名で絞り込み（部分一致） |
| ステータスセレクト | 選択 | ステータスで絞り込み |
| 各カラムヘッダー | クリック | 昇順/降順ソート |
| 行 | クリック | 案件詳細画面へ遷移 |
| 再送信ボタン | クリック | 店舗への依頼メールを再送信 |
| 三点メニュー（︙） | クリック | メニュー表示 |
| 新規サポート依頼ボタン | クリック | 案件作成モーダルを表示 |
| ページネーション | クリック | ページ切り替え |

---

## 🔄 状態パターン

| 状態 | 表示内容 |
| --- | --- |
| 初期表示 | ローディングスピナー |
| データあり | 一覧表示（1ページ10件） |
| データなし | 「案件が登録されていません。」 |
| 検索結果なし | 「該当する案件が見つかりません。」 |

---

## 🔐 権限

| ユーザー種別 | 閲覧 | 作成 | 編集 |
| --- | --- | --- | --- |
| 管理者 | ◯（全件） | ◯ | ◯ |
| オペレータ | ◯（全件） | ◯ | ◯ |
| パートナー | △（自店舗のみ） | ◯ | ◯ |
| コンシェルジュ | △（自分の案件のみ） | ◯ | ◯ |

※ パートナー・コンシェルジュは自分に関連する案件のみ表示・編集可能

---

## 三点メニュー

| 項目 | 動作 |
| --- | --- |
| 編集 | 案件編集モーダルを表示 |
| この案件のコピー | 案件作成モーダルを表示（内容をコピー） |

## ダイアログ

### 再送信確認

| 項目 | 内容 |
| --- | --- |
| タイトル | メール再送信 |
| メッセージ | 「○○（店舗名）に依頼メールを再送信します。よろしいですか？」 |
| ボタン | キャンセル / 再送信 |

## 📝 備考

- API: `GET /api/operator/cases`
- ステータス一覧：店舗未決定 / オペレーター確認中 / CS確認中 / 報告未完了 / 報告完了 等
- 再送信ボタンは店舗への依頼メールを再送
- 「この案件のコピー」はリピート案件の登録を効率化するための機能
