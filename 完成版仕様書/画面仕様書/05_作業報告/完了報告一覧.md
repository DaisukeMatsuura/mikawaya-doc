## 🎯 画面の目的

完了報告がまだ行われていない案件の一覧を表示し、報告画面へ遷移する。

---

## 📊 表示データ項目

| 項目名 | データ型 | 必須 | 備考 |
| --- | --- | --- | --- |
| 訪問日時 | DateTime | ◯ | ソート可能 |
| 案件No/お客様名 | String | ◯ | 2行表示 |
| ステータス | Enum | ◯ | バッジ表示（未報告） |

---

## ⚙️ 機能・操作

| 要素 | 操作 | 動作 |
| --- | --- | --- |
| 訪問日時カラムヘッダー | タップ | 昇順/降順ソート |
| 行 | タップ | 案件詳細画面へ遷移 |
| 報告ボタン | タップ | 完了報告画面へ遷移 |
| パンくずリスト（ホームアイコン） | タップ | ダッシュボードへ戻る |
| ページネーション | タップ | ページ切り替え |

---

## 🔄 状態パターン

| 状態 | 表示内容 |
| --- | --- |
| 初期表示 | ローディングスピナー |
| データあり | 一覧表示（1ページ10件） |
| データなし | 「未報告の案件はありません。」 |

## 📝 備考

- API: `GET /api/concierge/cases?status=unreported`
- スマホ専用画面（レスポンシブ対応）
- 自分が担当する案件のみ表示
- 「報告」ボタンから直接完了報告画面へ遷移できる
- 訪問日時でソート可能（デフォルトは古い順＝早く報告すべき順）
