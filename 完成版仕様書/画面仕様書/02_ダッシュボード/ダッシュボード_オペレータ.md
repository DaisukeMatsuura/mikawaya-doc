## 🎯 画面の目的

オペレーターが対応すべき案件の状況を一目で把握し、迅速に対応できるようにする。

---

## ⚙️ 機能・操作

| 要素 | 操作 | 動作 |
| --- | --- | --- |
| 新規サポート依頼ボタン | クリック | 案件作成モーダルを表示 |
| サマリーカード | クリック | 該当条件で絞り込んだ案件一覧へ遷移 |
| 担当店舗が未決定の行 | クリック | 案件詳細画面へ遷移 |
| 店舗が未確認の行 | クリック | 案件詳細画面へ遷移 |
| 再送信ボタン | クリック | 店舗への依頼メールを再送信 |
| 鉛筆アイコン | クリック | 案件編集モーダルを表示 |
| 一覧で見るリンク | クリック | 案件一覧画面へ遷移 |

---

## 🔄 状態パターン

| 状態 | 表示内容 |
| --- | --- |
| 初期表示 | ローディングスピナー |
| データあり | サマリー + 各一覧を表示（最大5件ずつ） |
| データなし | 「対応が必要な案件はありません。」 |

### 経過時間の警告表示

| 経過時間 | 表示 |
| --- | --- |
| 24時間以内 | 通常表示（グレー） |
| 24時間〜48時間 | 黄色警告 |
| 48時間以上 | 赤字警告（例：3日2時間3分） |

## 📝 備考

- API: `GET /api/operator/dashboard`
- 各一覧は最大5件表示、「一覧で見る」で全件表示の案件一覧へ遷移
- 再送信ボタンは店舗への依頼メールを再送（確認ダイアログあり）
- 経過時間は自動更新（1分ごと）または画面リロード時に更新
